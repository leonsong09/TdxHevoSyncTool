# user.ini 全板块定向同步设计

日期：2026-03-26

## 背景

`user.ini` 是通达信个性化配置核心文件，跨版本整体复制可能导致异常。当前工具已将其排除在普通导入/导出与双向对比之外，只提供单独的 `user.ini` 定向同步入口。

现有实现只处理 `extern_*` 段，而且采用“只补缺失键、不覆盖已有值”的语义。这与新需求不一致：新需求要求对所有 `[]` 板块生效，但采用更细粒度的规则，避免整段复制带来的风险。

## 目标

将源 `user.ini` 的设置按 section 粒度安全同步到目标 `user.ini`，满足以下要求：

1. 对所有 `[]` 板块生效。
2. 非 `extern_*` 板块：只替换目标中已存在且值不同的键，不追加缺失键。
3. `extern_*` 板块：既替换同名异值键，也追加目标中缺失的键。
4. 不新增任何段落。
5. 对重复 section、重复 key 等歧义结构采取阻止策略，而不是静默猜测。
6. 合并后文件结构保持可读、可回滚、可继续被通达信使用。

## 非目标

1. 不整体复制 `user.ini`。
2. 不新增目标中不存在的 section。
3. 不重写或整理所有 section 的顺序、注释和格式。
4. 不自动修复任意损坏的 `user.ini`。

## 选定方案

采用“按 section 感知的行级定向同步”方案：

1. 保留当前原始文本解析方式，继续兼容 UTF-8 BOM 和常见中文编码。
2. 在 section 内识别 `key=value` 行，建立轻量级 key 索引。
3. 非 `extern_*` section 只执行同名 key 的值替换。
4. `extern_*` section 在值替换之外，还允许追加缺失 key。
5. 目标缺失 section 时直接跳过，并在预览中明确提示。
6. 如果源或目标出现重复同名 section，或者目标 section 中存在重复 key，直接报错并阻止写回。

未采用方案：

1. 使用标准 INI 解析器整体读写。
原因：容易破坏原始注释、顺序、大小写和编码，对历史配置文件风险偏高。

2. 整段覆盖同名 section。
原因：会误删目标中未被源覆盖但仍有意义的内容，也会丢失局部格式与注释。

3. 自动补出目标中不存在的普通 section。
原因：用户已明确要求不新增段落，应保持最小影响范围。

## 设计细节

### 解析

1. 读取 `user.ini` 时继续兼容 UTF-8 BOM。
2. 首个节头前的内容仍保留在 `header_lines`。
3. `extern_*` 匹配保持大小写不敏感。
4. section 名称匹配采用大小写不敏感。
5. section 内只把包含首个 `=` 的行识别为配置项，其余注释或空行原样保留。

### 合并规则

对于每个源 section：

1. 若目标不存在同名 section：
   - 不新增段落。
   - 记录为“缺失 section，跳过”。
2. 若目标存在且仅存在一个同名 section：
   - 对目标和源分别建立 key 索引。
   - 若目标 section 存在重复 key：直接报错。
   - 若源 section 存在重复 key：仅保留第一次出现的值，后续重复项忽略。
   - 对同名 key：
     - 若值相同，跳过。
     - 若值不同，用源行替换目标行。
   - 对仅存在于源的 key：
     - 若 section 为 `extern_*`，追加到目标 section 末尾。
     - 否则跳过。
3. 若源或目标存在多个同名 section：
   - 直接报错。
   - 不写回目标文件。

### 写回规则

1. 写回前必须先创建 `.ini.bak` 备份。
2. 替换时尽量原位替换目标 key 行，不重排 section 中其他内容。
3. 对 `extern_*` 追加 key 时，若目标段末行无换行，先补足换行。
4. 不重排 section 顺序，不整理注释，不新增 section。
5. 尽量保持目标文件原始编码；编码写回失败时按现有回退策略处理。

### UI 行为

1. 对话框文案从“extern 合并”升级为“user.ini 定向同步”。
2. 列表显示源和目标的全部 section，不再只显示 `extern_*`。
3. 预览阶段应区分以下结果：
   - 将替换键值
   - `extern_*` 将追加键值
   - 目标缺失段落，已跳过
   - 无需变更
4. 预览阶段如果发现重复 section 或重复 key，应直接提示错误，不允许继续确认同步。
5. 成功后仍提示 `.ini.bak` 已生成。

## 测试策略

本次变更采用 TDD，至少覆盖以下回归测试：

1. UTF-8 BOM 文件中的首个 section 能被识别。
2. 非 `extern_*` section 中，同名异值 key 会替换。
3. 非 `extern_*` section 中，源独有 key 不会追加。
4. `extern_*` section 中，同名异值 key 会替换，缺失 key 会追加。
5. 目标缺失 section 时不会新增段落。
6. 源或目标存在重复同名 section 时，预览或应用阶段报错。
7. 目标 section 内存在重复 key 时，预览或应用阶段报错。
8. 目标段末行无换行时，`extern_*` 追加后的结果仍是合法换行分隔。

## 影响范围

主要影响：

1. `src/core/userini_handler.py`
2. `src/ui/userini_dialog.py`
3. `tests/test_userini_handler.py`
4. `tests/test_userini_dialog.py`
5. README 与界面提示文案

不影响：

1. 普通导出/导入逻辑
2. 双向对比/同步主流程
3. `user.ini` 禁止整体迁移这一产品规则

## 验收标准

满足以下条件即可认为完成：

1. 所有 section 的同名异值 key 能按规则替换。
2. 只有 `extern_*` section 会追加缺失 key。
3. 缺失 section 不会被新增。
4. 重复 section 或重复 key 不会被静默放行或写坏。
5. 相关测试、UI 文案、README 与发布产物说明保持一致。
